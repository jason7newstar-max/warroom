# 计划书 · MAESTRO
### 用语言作曲 × 用手势指挥的实时交响表演乐器
### A Language-Composed, Gesture-Conducted Live Orchestral Performance Instrument

> 项目代号(暂定):**MAESTRO**　|　War Room ID: **T-010**
> 起草:IA10(ONE TEN War Room）· 主创/总监:文天(科班作曲)· 2026-06-05

---

## 0. 一句话
**用自然语言精确地"写"出一部可编辑的交响总谱,再用摄像头捕捉的指挥手势在观众面前"演"出它** —— 把正在被 AI 贬值的"创作",重新锚定在 AI 难以替代的"现场表演"上。

---

## 1. 出发点 / 战略论点
- AI 让"生成内容"趋近于零成本,纯创作端(及其版权)正在被重新定义、被贬值。
- 未来十年最难被替代的,是**真人表演者** —— 现场、即时、不可复制的"人"。
- 因此本项目不止于"作曲软件",而是一个**现场表演乐器**:创作过程本身成为表演,人始终在环路中心。

## 2. 现状与缺口(为什么现在做、为什么是我们)
把已有工作分四类(详见 `LANDSCAPE.md`):
1. **黑箱 text→MIDI 生成器**(MIDI-LLM / Text2midi / MuseCoco):质量高但 prompt 完听天由命,**无逐音/逐小节控制**。
2. **多 agent 作曲**(ComposerX / CoComposer):把"主题→和声→配器"做成角色 agent,**思路对但一次性、不可编辑**。
3. **语言改音符 copilot**(FL Studio music-copilot / Ableton Copilot):交互接近,但绑 DAW、面向流行,**非管弦总谱级**。
4. **手势指挥系统**(Google Semi-Conductor / MediaPipe / arXiv 2604.27957):成熟,但**只指挥一段固定录音**(改 playback 速度/音量)。

**缺口 = 我们的护城河:** 没有人把"**作曲家思维 + 总谱原生 + 增量可编辑 + 语言精确控制**"和"**实时手势指挥一套活谱**"融合成一个表演乐器。现有指挥 demo 指挥的是死录音;我们指挥的是**用语言现场写出来、可改的活谱**。

## 3. 产品概念 —— 语言 + 手势,双输入

> **优先级:先把核心技术做出来(作曲引擎)。Live 演出是后话(第二阶段)。** 但"手势"从一开始就不只属于演出 —— 它也是**作曲过程里的一个输入通道**。

### 第一幕 · 作曲(语言 + 手势)
你用自然语言搭总谱:
> "第1小节,4/4,弱起;小提琴 I 拉 A4 二分音符,叠一个 A 大三和弦,再叠成大七和弦;大提琴走根音。"

**手势作为作曲辅助(关键洞察):** 古典作曲里,速度/节奏一直是用**抽象的数字**描述的(♩=88)。但"我心里想要的那个速度"很难用数字精确表达。所以 —— **用手的快慢直接"划"出速度和节奏型**:
- 手速 → 直接定这首/这段的 tempo(比敲一个数字精确得多,是"我要的那个速度")。
- 手势的律动 → 描述某声部的**节奏型**(比如小提琴该怎么走的快慢/顿挫)。
- 手势 = 一个**实时、交互、帮你作曲**的辅助通道,把抽象的数字变成身体直觉。

系统把这一切落成一份**结构化总谱(核定版)**,可看(五线谱)、可听(音源)、可随时改任意一小节/声部。

### 第二幕 · 表演(指挥)— 后话 / 第二阶段
等核心引擎跑通,再把同一套手势能力升级成**舞台指挥**:摄像头捕捉指挥动作,实时驱动这套活谱在观众面前发声 —— 挥拍→tempo、手幅→力度、cue→声部进入、左手→表情。最终收束到"核定版 + 现场即兴"的演出。**这是远期,不是 M0。**

## 4. 技术架构

```
 ┌──────────────────────── 第一幕:作曲 ────────────────────────┐
   你的话(文字 / 语音)
        │
        ▼
   [LLM 解析器]  不直接写音符,只翻译成"原子操作"(可验证)
        │   add_note(part,m,beat,pitch,dur) / stack_chord(quality) /
        │   set_tempo / set_meter / anacrusis / edit_measure / transpose …
        ▼
   [结构化总谱 = music21 Score]   ← 唯一真相,JSON/MusicXML 可版本化(git)
        │
        ├─► MusicXML ─► [五线谱渲染 Verovio / LilyPond]
        └─► MIDI ─────► [音源 FluidSynth + 管弦 SoundFont]
 └──────────────────────────────────────────────────────────────┘
                          │  核定版活谱
                          ▼
 ┌──────────────────────── 第二幕:指挥 ────────────────────────┐
   [摄像头] ─► [姿态识别 MediaPipe Pose / PoseNet] ─► [手势→音乐映射]
        挥拍→tempo · 手幅→力度 · cue→声部进入 · 左手→表情
        │
        ▼
   [实时渲染引擎]  跟手出声(MIDI/采样) + 谱面跟随 + 可投影视觉
 └──────────────────────────────────────────────────────────────┘
```

**核心设计原则:** LLM 只产出*确定的、可验证的编辑操作*,谱子由 music21 持有。⇒ 控制度极高、可增量编辑、可回滚分支、解析层与渲染层解耦(模型随时可换)。

## 5. 技术选型表(什么技术做什么)

| 模块 | 选用技术 | 角色 / 为什么 |
|------|----------|---------------|
| 语言解析 | LLM(Claude / GPT / 本地 Gemma)+ 受约束输出(JSON schema / 函数调用) | 把自然语言 → 确定的"原子操作",不让它瞎生成音符 |
| 总谱内核 | **music21**(Python, BSD) | 业界标准的符号乐谱模型;原生 MusicXML + MIDI 导出;和弦/调性/声部对象齐全 |
| 中间表示 | 自定义 JSON DSL(+ 可选 ABC notation) | 可版本化、可 diff、LLM 友好;ABC 备选做紧凑交换 |
| 乐谱显示 | **Verovio**(MusicXML→SVG, 浏览器)/ LilyPond(高质量制谱) | 实时看五线谱;LilyPond 出印刷级总谱 |
| 音频出声 | **FluidSynth** + 管弦 SoundFont(.sf2);后期可上采样库 | 免费、快,先验证;后期换更真的音色 |
| 指挥捕捉 | **MediaPipe Pose Landmarker** / PoseNet(浏览器, 无需硬件) | 鲁棒的上半身2D姿态,适合公开演出 |
| 手势→音乐 | 挥拍检测 + 速度/力度映射(参考 LSTM/DTW 手势识别) | 把动作实时翻译成 tempo/力度/cue |
| 语音输入(选配) | 复用现有 voice-stt(Groq Whisper) | 开车/灵感来时直接"说"谱 |
| 前端/演出 | Web(JS + WebAudio/WebMIDI)或本地 app | 浏览器即可演;低门槛、可投影 |

## 6. 路线图(里程碑)

| 里程碑 | 内容 | 验证目标 |
|--------|------|----------|
| **M0 闭环** | 命令行:一句话 → music21 → 导出 .mid + FluidSynth 出声 | 证明"语言→可控音符"通 |
| **M1 增量** | 多声部、和弦叠加、按小节编辑、save/load 谱(JSON) | 证明"可控可编辑" |
| **M2 看得见** | 网页 + Verovio 实时五线谱 | 作曲家能看着谱改 |
| **M2.5 手势定速** | 摄像头→MediaPipe,**手速→tempo / 节奏型**(作曲辅助,非演出) | 证明手势能精确"划"出速度 |
| **M3 指挥(后话)** | 升级成舞台指挥:挥拍→tempo、手幅→力度 实时控活谱 | 第二阶段·演出 |
| **M4 演出(后话)** | cue 声部进入 + 表情 + 投影视觉,完整 live | 可上台 |
| **M5 拓展** | 流行 / 乐队 / 电音版本 | 复用引擎换音色/编制 |

> 顺序:**先把核心作曲引擎(M0–M2)做出来**,手势先作为作曲辅助(M2.5),**Live 演出(M3–M4)是后话**。乐种:先交响乐 → 后拓流行/电音。

## 7. 与现有工作的差异化(护城河)
- vs 生成器:我们**可控、可编辑、可版本化**,不是黑箱。
- vs 多 agent 作曲:我们**交互式、活谱、能反复改**,不是一次性。
- vs DAW copilot:我们**总谱原生、管弦级、作曲家思维**,不绑钢琴卷帘。
- vs 指挥 demo:我们指挥的是**语言现场写出的活谱**,不是死录音。
- **唯一的融合**:作曲(语言)× 指挥(手势)× AI 乐团 × 真人表演者。

## 8. 风险与对策
- **LLM 音乐准确性** → 用"受约束的原子操作 + music21 校验",不让模型直接吐音符。
- **手势→音乐的实时延迟/抖动** → 先低分辨率映射(只控 tempo/力度),滤波平滑;参考成熟 LSTM/DTW。
- **音色真实度** → 分期:先 SoundFont 验证思路,后期采样库。
- **范围蔓延** → 严格里程碑,M0 命令行先跑通再谈演出。
- **版权/法律** → 输出是用户自创总谱(MusicXML),归属清晰;避开训练数据版权争议。
