# T-022 · Scan homework → auto-generate a video game (edtech)

**Spark:** Chairman 2026-06-06 (saw a demo). Scan a homework worksheet → AI auto-builds a playable video game from it (gamified practice).

## Feasible NOW — but the realistic architecture matters
- **Don't** auto-generate a novel game ENGINE each time (unreliable, breaks). **Do**: OCR/vision reads the worksheet → extract the problems/concepts (math, vocab, etc.) → **inject that content into pre-built game TEMPLATES** (quiz-platformer, math-runner, vocab-match, etc.). Reliable + fast + fun-by-design.
- Pieces all exist: vision OCR (read worksheet), LLM (parse problems + generate distractors/levels), a small library of templated web games (Phaser/web). Chairman's visual machine can make the games LOOK great = an edge.

## Honest reality
- **Not novel** — AI "content→game" + edu-gamification tools already exist (MagicSchool, Roblox edu, various). Wedge must be: the INSTANT scan→playable flow + game quality + a sharp subject/age niche.
- **The hard part is NOT the tech** — it's edtech distribution + monetization + retention: who pays (parents D2C vs teachers vs schools — schools = long cycles), and whether auto-games stay fun/pedagogically sound past novelty.
- **Best wedge:** a viral CONSUMER app — parent scans kid's homework → instant fun practice game in 10s — riding the visual polish. Pick ONE subject (e.g. elementary math) to nail first.
